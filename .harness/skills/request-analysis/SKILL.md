# 需求分析技能

把需求 / 任务包变成可执行的缺口清单与设计。

## 何时用
开新特性、对齐原型、产品给了任务包但口径不清时。

## 步骤
1. **读源**：`D6_任务包` 原文 + 相关 `wiki/`（业务 / 接口 / 数据模型）+ 现有代码。
2. **三方对账**：需求文字 × 原型（注意对的是哪一版原型）× 现有代码三层，逐条核。
3. **分缺口**：区分「真缺口」（三层全无）与「口径偏差」（实现了但口径不符）。
4. **落清单**：缺口写成 md + 任务列表；大任务在 `../../changes/` 建变更记录。
5. **确认设计点**：单个缺口内的关键设计点逐点和人确认，忠于需求原文，别自作主张。

## 产出
缺口清单 / 设计方案 → `docs/superpowers/specs/` 或 `../../changes/`。
