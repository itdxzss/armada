# 数据模型规范

armada 库表设计与变更规范。**根治"没看懂全局设计就加列加表"导致的 schema 熵增。**

> 全局现状见 `wiki/数据模型.md`(自动生成,改表后重跑 `wiki/gen_datamodel.py`,**禁手改**)。

## 一、反 schema 屎山(改表前必读,最高优先级)

1. **加列/加表前先看全局**:动手前先读 `数据模型.md` 全局模型,并在设计文档里答清——这列/表**属于哪个聚合、为什么必须加、为什么现有列/表不行、会不会与现有重复**。答不上就别加。
2. **一个事实一处存(禁分歧)**:同一概念只用**一列/一张表**表达,**禁止并行多套表示**(反面教材:账号分组的 `account_group_id` / `tag_id` / `account_tag` 三镜像,会漂移)。新建前先确认没有等价的已存在。
3. **宽表垂直拆分**:单表列数过多(经验阈值 **> ~30 列**)或混入不相关关注点时,按聚合拆表(如 `account` 核心 / `account_risk` 风控 / `account_proxy` IP 绑定)。**一张表 = 一个聚合**,不做 grab-bag。
4. **禁死列/死表**:不为"将来可能用"预留列;加的列必须有人写、有人读;确认废弃的列/表用 Flyway `DROP`,不留 NULL 死列(反面教材:`account.ip_type` 67 行全 NULL)。
5. **表数克制(双向)**:能在现有聚合加一列解决的,不另起一张表;但不相关的关注点也不许堆进同一张宽表。
6. **改 schema 必更新文档**:重跑 `gen_datamodel.py` 刷新 `数据模型.md`;新列必须带 `COMMENT` 注明业务含义(自动文档靠它)。

## 二、命名与通用列(沿用现状,统一)

- 表名:**单数 snake_case**(`account`、`group_link`、`op_log_tenant`)。
- 列名:snake_case;布尔 `is_xxx`;时间 `xxx_at`;外键 `xxx_id`。
- 主键:`id` BIGINT。
- 时间戳:`created_at` / `updated_at`(**禁** `create_time`/`update_time`)。
- 软删:`deleted_at`(NULL = 未删;**禁** `is_deleted`)。
- 状态/枚举列:`TINYINT`,**逐值 `COMMENT` 说明业务含义**;Java 侧映射 enum / 常量,禁魔法值(呼应编码规范红线 #1)。

## 三、多租户隔离

- 租户级表**必须有** `tenant_id`,租户拦截器自动注入 `WHERE tenant_id=?`。
- **无 `tenant_id` 的表**(平台元数据表、靠 `account_id` 等关联的关系表)**必须登记进** `platform/persistence` 的 `TenantLineConfig.IGNORED_TABLES` 白名单,否则被注入 `AND tenant_id=?` → `Unknown column 'tenant_id'` 崩库。
- 跨租户访问只在 `admin` 域用 `@IgnoreTenant`(声明非空 reason)。

## 四、Flyway 迁移

- 所有 schema 变更走 Flyway:`armada-api/src/main/resources/db/migration/V<nnn>__<snake_desc>.sql`;**禁手动 ALTER 共享库**(绕过 Flyway → checksum/撞列 → 启动 crash-loop 全站挂)。
- 版本号 `V<nnn>` 全局唯一,跨分支提交前**核对防撞号**。
- `ADD COLUMN` 前用 `information_schema` 守卫保证幂等(列已存在则跳过)。
- 改 mapper XML 配套时先 `xmllint` / 真库 DbTest(裸 `<>` 运行时炸)。
- 变更随附数据迁移 + 回滚脚本:`.harness/changes/<业务>/db-migrations.sql` + `rollback.sql`。

## 五、本次 armada 数据模型重设计(野心:重)

- **目标**:合并冗余表、拆分宽表、归一分歧(三镜像)、清死列 → 产出**目标模型**。
- **落地**:per-business 重建时把该业务的表对齐目标模型 + 数据迁移脚本。
- **跨业务共享表(如 `account`)的任何改动走全局评审**,禁某业务私自加列。
- 红线:重设计后每张表都要能在设计文档里说清它的**聚合归属**与**每列必要性**。
