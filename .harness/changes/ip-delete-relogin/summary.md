# IP 删除在线账号重登

## 变更概述
- IP 批量删除改为先执行删除编排:对即将删除 IP 当前绑定的在线账号发起换 IP 重登,再软删 IP。
- 离线账号或无登录态账号不触发重登,保持原绑定数据不额外处理。
- 代理分配新增排除列表,避免重登时刚释放的待删除 IP 又被重新分配。
- `IpProxyService` 不再暴露批量删除方法,避免业务入口绕过删除编排。

## 影响模块
- `resource`:IP 删除入口、删除编排服务、代理分配与绑定查询 Mapper。
- `account`:新增按待删 IP 触发在线账号重登的服务方法。

## 数据库变更
- 无表结构变更。
- 新增 MyBatis 查询:
  - 查询待删代理当前绑定账号。
  - 锁定空闲代理时排除指定代理 ID。
  - 按账号 ID 筛选 `login_state=ONLINE` 账号。

## API 变更
- `POST /api/ip-proxies/batch-delete` 入参/出参不变。
- 行为变更:若待删 IP 绑定在线账号,会先写入账号上线 outbox 以换 IP 重登;重登编排失败则不软删 IP。

## Redis 变更
- 无。

## 关键约束
- 离线账号不动。
- 在线账号必须排除本次删除的 IP 重新分配代理。
- 删除接口只保证上线命令进入 outbox,真实重登结果仍由协议层异步状态回写。

## 回滚方案
- 回退本次代码即可恢复为直接软删 IP 的行为。
