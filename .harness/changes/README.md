# 变更记录（changes/）

> 跨会话可恢复的进度与决策记录。这是 Harness「持久化记忆 + 结构化执行」的落地：
> 让任何 Agent 接手时能从文件恢复上下文，不靠会话记忆。

## 何时建
多步骤 / 大功能 / 跨会话的任务。小修小补不用建。

## 怎么用
1. 开工前 `cp _TEMPLATE.md YYYY-MM-DD-<主题>.md`，填「目标」+「缺口拆解」。
2. 推进中持续更新「任务清单」「关键设计决策」「验证」。
3. 完成后补「部署」「遗留」，**保留文件作历史**（不删）。

## 命名
`YYYY-MM-DD-<kebab-主题>.md`，例：`2026-06-16-pull-task-detail-stats.md`。

> 注：`docs/_tasks/` 是更早的零散计划区；新任务统一用本目录的模板，逐步收口。
