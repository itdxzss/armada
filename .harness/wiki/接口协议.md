# 一、公开 API
> 公开路径,通常不需要租户上下文。

## 租户登录 API（TenantAuthController）

### POST /api/public/auth/login

租户登录:校验租户码 + 密码,成功返回租户信息与占位 token。

- 权限：暂无接口级权限注解
- 请求体：`TenantLoginRequest`（登录入参(租户码 + 密码)）
- 响应 `data`：`TenantLoginVO`（登录结果(租户码、租户名、占位 token)）

# 二、租户业务 API
> 租户侧业务接口;租户上下文由 armada 后端统一处理。

## 账号分组 API（AccountGroupController）

### GET /api/account-groups

账号分组列表(分页 + 关键字筛选)。 同时懒创建系统默认分组(Service 层保证幂等)。

| 参数 | 位置 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|------|
| query | query | AccountGroupQuery | 否 | - | 查询条件 |

- 权限：暂无接口级权限注解
- 响应 `data`：`分页（AccountGroupVO）`（分页分组列表）

### POST /api/account-groups

新建账号分组。

- 权限：暂无接口级权限注解
- 请求体：`AccountGroupDTO`（分组信息）
- 响应 `data`：`AccountGroupVO`（新建/复活的分组 VO）

### POST /api/account-groups/batch-delete

批量软删除账号分组(全或无闸门:组内有账号或命中系统组则整批拒删)。

- 权限：暂无接口级权限注解
- 请求体：`AccountIdsDTO`（分组 ID 列表）
- 响应 `data`：`Integer`（实际删除数）

### PUT /api/account-groups/{id}

修改账号分组基本信息(系统默认分组不可改名)。

| 参数 | 位置 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|------|
| id | path | Long | 是 | - | 路径中的分组 ID |

- 权限：暂无接口级权限注解
- 请求体：`AccountGroupDTO`（更新内容）
- 响应 `data`：无（仅 `code` / `message`）

## IP 代理 API（IpProxyController）

### GET /api/ip-proxies

分页查询 IP 代理列表：按国家 / 类型 / 来源 / 关键字组合筛选并分页，供列表页展示。

| 参数 | 位置 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|------|
| query | query | IpProxyQuery | 否 | - | 搜索与分页条件 |

- 权限：暂无接口级权限注解
- 响应 `data`：`分页（IpProxyVO）`（当前页代理及总数）

### POST /api/ip-proxies/batch-delete

批量删除 IP 代理（软删）。

- 权限：暂无接口级权限注解
- 请求体：`IpProxyBatchDeleteDTO`（选中的代理 ID 列表）
- 响应 `data`：无（仅 `code` / `message`）

### POST /api/ip-proxies/import

批量导入 IP 代理：逐行解析 host:port:用户名:密码，去重 + 格式校验后入库，返回导入统计。

- 权限：暂无接口级权限注解
- 请求体：`IpProxyImportDTO`（导入参数（国家、协议、来源、多行原文））
- 响应 `data`：`IpProxyImportResultVO`（导入结果）
