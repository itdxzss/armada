## 1. 范围

**本块做(import core,纯 DB、零协议耦合)**:
- WS链接分组(`group_link_label`)CRUD:新增(名称唯一+使用国家+备注)/编辑/批量删除(级联软删)
- 导入群链接:text ∪ 文件(TXT/CSV/Excel)拼接 → 格式校验 + 去重 → 落 `group_link` + 批次/明细
- 明细(行号/群名/链接/来源文件/导入状态/失败原因/时间)、导出失败数据
- 迁移分组、列表搜索(分组维度,「建分组即出行」)

**本块不做(拆出去,协议耦合,随对应块 + 协议层改造落地)**:
- 链接群组检测(preview 解析 group_jid/群名/人数/封禁 + 健康巡检)→ `group_link_preview` / `group_link_health` 表(见 §5.1 目标形态、§9 延后 seam)
- 群组列表的群信息抽屉、成员操作、群设置(属群组列表块)

**为什么先做这块**:它是 wheel↔协议层改造在途时唯一零协议耦合、可独立验收的纵切;且是进群/拉群的群链接池根依赖。详见记忆 `armada_block_sequencing_0624`。

## 3. 现状审计(wheel,作"血泪规格书"非 copy 源)

- 逻辑埋在 `TenantBusinessBasicsService`(1700+ 行上帝类,混 tag/group-link/material/marketing)→ armada 刨成 `group` 域独立 service。

- `group_link` 是 28 列宽表,混了 import 身份 / 检测群元数据 / 健康运行态三类关注点 → armada 按 `数据模型规范一.3` 垂直拆分(见 §5.1)。

## 5. 数据模型

### 5.1 `group_link` 全局目标形态(跨业务共享表,对齐)

`group_link` 是被 **导入链接 / 拉群 / 进群 三入口写 + 群组列表读** 的共享群组表。按 `数据模型规范一.3`(>~30列宽表按聚合垂直拆分,如 account→account/state/risk)+ `五`(跨业务共享表全局评审),目标形态拆 **3 张表**(1:1 关联):

| 表 | 列(聚合) | 归属块 | 本期建? |
|---|---|---|---|
| **`group_link`**(import 身份+归属) | link_url, group_name, label_id, import_batch_id, remark, +通用列;**[后续块加]** origin(来源)、membership_state(TARGET/JOINED) | 导入链接 | ✅(见 5.2,后续列由对应块经评审加) |
| **`group_link_preview`**(协议群元数据,1:1) | group_jid, wa_subject, member_size, invite_code, owner_phone, announce_only, avatar_url, last_preview_at | 检测/群组列表 | ⏸ 延后 |
| **`group_link_health`**(健康+运行态,1:1,含 **is_banned 收敛锚**) | health_status, is_banned, current_count, last_check_at, last_health_error, health_failure_count | 检测/巡检 | ⏸ 延后 |

> **全局评审已在此完成**:上表即 group_link 家族的目标形态,后续块**只能按此加表/加列**(`数据模型规范五`:禁某业务私自加),不得重走 wheel 28 列宽表。
> `is_banned` 是跨派单/营销/进群的**共享可用性收敛锚**(它们只读这一列判可用),语义须全局定 → 归 health 表、延后到检测块。
> wheel 死列 `capacity` / `expire_at`(五业务零读写)**armada 直接不建**。

### 5.2 本期建的 4 张表(逐表:聚合归属 + 列必要性 + DDL)

> 命名:表单数 snake_case;时间列 `*_at`(DATETIME/UTC);软删 `deleted_at`;枚举列 TINYINT + 逐值 COMMENT + Java enum(禁魔法值)。
> `xxx_id` 均为**关联列(查询 JOIN,不建物理外键)**。

#### 表1 `group_link_label` — WS链接分组
**聚合**:WS链接分组(导入链接列表行,「建分组即出行」)。**唯一**:租户内 name 不重复。
```sql
CREATE TABLE IF NOT EXISTS group_link_label (
    id          BIGINT       NOT NULL AUTO_INCREMENT                     COMMENT '主键',
    tenant_id   BIGINT       NOT NULL                                    COMMENT '租户ID(拦截器注入)',
    name        VARCHAR(100) NOT NULL                                    COMMENT 'WS链接分组名称(租户内不可重复)',
    region      VARCHAR(64)           DEFAULT NULL                       COMMENT '使用国家/区域展示名(可「混合」)',
    remark      VARCHAR(255)          DEFAULT NULL                       COMMENT '备注',
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP                            COMMENT '创建时间(UTC)',
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间(UTC)',
    created_by  BIGINT                DEFAULT NULL                       COMMENT '创建人user_id',
    deleted_at  DATETIME              DEFAULT NULL                       COMMENT '软删时间;NULL=未删',
    PRIMARY KEY (id),
    UNIQUE KEY uq_name (tenant_id, name),
    KEY idx_tenant_created (tenant_id, created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='WS链接分组';
```
- 列必要性:`name` 列表主标识+唯一校验;`region` 需求「使用国家」;`remark` 需求备注;通用列标配。
- 软删与重名:新增时活的同名→报「名称重复」;软删的同名→复活该行(`deleted_at=NULL`),不插新行(plain 唯一键 + 复活)。

#### 表2 `group_link` — 群链接(共享群组表 · import 身份段)
**聚合**:群链接活池(迁移/编辑/下游任务读;一条 url 一行)。**唯一**:租户内 link_url。
```sql
CREATE TABLE IF NOT EXISTS group_link (
    id              BIGINT       NOT NULL AUTO_INCREMENT                 COMMENT '主键',
    tenant_id       BIGINT       NOT NULL                               COMMENT '租户ID(拦截器注入)',
    link_url        VARCHAR(255) CHARACTER SET utf8mb4 COLLATE utf8mb4_bin NOT NULL COMMENT '归一化后群邀请链接(租户内唯一,按字节精确去重)',
    group_name      VARCHAR(128)          DEFAULT NULL                  COMMENT '业务群名(导入时填,可空;协议官方名wa_subject在preview表)',
    label_id        BIGINT                DEFAULT NULL                  COMMENT '所属WS链接分组(关联group_link_label.id;只导入链接菜单写;拉群/进群粘贴=NULL)',
    import_batch_id BIGINT                DEFAULT NULL                  COMMENT '来源导入批次(关联group_link_import_batch.id;编辑视图「来源文件」溯源)',
    remark          VARCHAR(255)          DEFAULT NULL                  COMMENT '备注(纯导入备注,不被健康复用)',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP                            COMMENT '创建时间(UTC)',
    updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间(UTC)',
    created_by      BIGINT                DEFAULT NULL                  COMMENT '创建人user_id',
    deleted_at      DATETIME              DEFAULT NULL                  COMMENT '软删时间;NULL=未删',
    PRIMARY KEY (id),
    UNIQUE KEY uq_url (tenant_id, link_url),
    KEY idx_tenant_label (tenant_id, label_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='群链接(跨业务共享群组表-import身份段)';
```
- 列必要性:`link_url` 核心标识+唯一去重锚;`group_name` 需求群名;`label_id` 归属(可空,见 §5.3);`import_batch_id` 编辑视图来源文件;`remark` 备注。
- **检测/health 列(group_jid/member_size/is_banned/health_status…)本期不建**(禁死列;由 preview/health 表承载,见 §5.1)。
- **origin/membership_state 本期不建**:本块不读它们(`来源` 列属群组列表、`JOINED` 由进群推进),按禁死列待对应块经全局评审加 + 回填(现有 import 行回填 origin=IMPORT、membership_state=TARGET)。

#### 表3 `group_link_import_batch` — 导入批次(明细头)
**聚合**:一次导入操作=一行(来源文件/批次名/计数/时间)。被明细 JOIN 读(来源文件名+导入时间)。
```sql
CREATE TABLE IF NOT EXISTS group_link_import_batch (
    id               BIGINT       NOT NULL AUTO_INCREMENT               COMMENT '主键',
    tenant_id        BIGINT       NOT NULL                             COMMENT '租户ID(拦截器注入)',
    label_id         BIGINT       NOT NULL                             COMMENT '导入目标WS链接分组(关联;导入时固定,不随迁移变)',
    batch_name       VARCHAR(255) NOT NULL                             COMMENT '来源文件/批次名称(用户填)',
    source_file_name VARCHAR(255)          DEFAULT NULL                COMMENT '上传文件原名;纯text导入=NULL',
    total_rows       INT          NOT NULL DEFAULT 0                   COMMENT '解析总行数',
    inserted_rows    INT          NOT NULL DEFAULT 0                   COMMENT '新增行数',
    adopted_rows     INT          NOT NULL DEFAULT 0                   COMMENT '收编行数(已存在url归到本分组)',
    skipped_rows     INT          NOT NULL DEFAULT 0                   COMMENT '批内重复跳过行数',
    failed_rows      INT          NOT NULL DEFAULT 0                   COMMENT '格式不合格行数',
    created_at       DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP                COMMENT '导入时间(UTC)',
    created_by       BIGINT                DEFAULT NULL                COMMENT '创建人user_id',
    deleted_at       DATETIME              DEFAULT NULL                COMMENT '软删时间;随分组级联软删',
    PRIMARY KEY (id),
    KEY idx_tenant_label (tenant_id, label_id),
    KEY idx_tenant_created (tenant_id, created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='群链接导入批次';
```

#### 表4 `group_link_import_detail` — 导入明细(逐行结果)
**聚合**:逐行导入审计(全部行,含失败);明细 + 导出失败数据源。不软删(审计不可变,随 batch 过滤)。
```sql
CREATE TABLE IF NOT EXISTS group_link_import_detail (
    id            BIGINT       NOT NULL AUTO_INCREMENT                  COMMENT '主键',
    tenant_id     BIGINT       NOT NULL                                COMMENT '租户ID(拦截器注入)',
    batch_id      BIGINT       NOT NULL                                COMMENT '所属批次(关联;分组维度经JOIN batch.label_id取)',
    line_no       INT          NOT NULL                                COMMENT '拼接后行号(明细「行号」列)',
    raw_url       VARCHAR(255)          DEFAULT NULL                   COMMENT '原文链接(失败行也保留原文)',
    group_name    VARCHAR(128)          DEFAULT NULL                   COMMENT '群名称(可空)',
    result        TINYINT      NOT NULL                                COMMENT '导入结果:1=成功新增 2=收编 3=批内重复 4=格式错误',
    fail_reason   VARCHAR(255)          DEFAULT NULL                   COMMENT '失败原因(result≥3时;如「格式错误」「重复」)',
    group_link_id BIGINT                DEFAULT NULL                   COMMENT '成功/收编时关联group_link.id;失败=NULL',
    created_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP                 COMMENT '创建时间(UTC)',
    PRIMARY KEY (id),
    KEY idx_tenant_batch (tenant_id, batch_id),
    KEY idx_tenant_batch_result (tenant_id, batch_id, result)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='群链接导入明细';
```
- `result` 配 Java enum `GroupLinkImportResult`(1 SUCCESS / 2 ADOPTED / 3 DUPLICATE / 4 FORMAT_ERROR),逐值 Javadoc。

> **`group_link_history`(收编/迁移审计)本期不建**:无读者(sheet 5 无审计视图)→ 按禁死列延后;归属变更暂由 `detail.result=ADOPTED` + batch 计数 + 日志兜底。需要归属变更审计视图时二期补(开放点 §13)。

### 5.3 共享表语义约定(铁律)

1. **归属铁律**:`label_id`(WS分组归属)是**导入链接菜单的独占写域**。进群/拉群只「找到或新建」该行来用,**绝不写/改 label_id**;它们新建的链接 `label_id=NULL`。
   - 场景:导入链接把 X 导进分组A → 进群任务再导入 X → X **仍属分组A**(进群只复用,不改归属)。
2. **entry-point-aware upsert(按 url)**:
   - 新 url → 插入。
   - 已存在(活/软删均按 url 命中)→ 复活(`deleted_at=NULL`)+ 收编(改 label_id);**仅导入链接/迁移入口触发收编**。
   - 批内重复(同文件出现两次)→ 跳过(DUPLICATE)。
3. **三正交维度**:归属(`label_id`,导入链接写,可迁移)/ 来源(`origin`,首建即定不可变,群组列表读)/ 使用(进群拉群任务行表持 `group_link_id`,不影响归属)。一条链接可同时属一个分组 + 被多任务用。
4. **完整性走应用层**:级联软删(删分组→Service 依次软删 links→batches)、归属/租户校验在 Service;不靠 DB 外键/cascade。

## 6. 端点(10 个,`com.armada.group`,kebab 路径,统一 `ApiResponse<T>`/`PageResult<T>`,时间出参 epoch 毫秒)

### A. WS链接分组
- **A1** `GET /api/group-link-labels` — 分组列表
  入参 Query(@ModelAttribute extends PageQuery):`keyword`(名称模糊)/`id`/`created_from`/`created_to`/`page`/`page_size`
  出参 `PageResult<GroupLinkLabelVO>`:`{id, name, region, remark, link_count(子查询聚合活链接数), created_at, updated_at}`
- **A2** `POST /api/group-link-labels` — 新增分组
  入参 DTO:`{name, region, remark}`;校验 name 唯一(活同名报「名称重复」,软删同名复活)
  出参 `ApiResponse<GroupLinkLabelVO>`(回显供导入框选)
- **A3** `PATCH /api/group-link-labels/{id}` — 编辑分组 `{name, region, remark}` → `ApiResponse<Void>`
- **A4** `POST /api/group-link-labels/batch-delete` — 批量删分组
  入参 `{ids:[]}`;占用闸(task 表未建→延后 no-op)→ 级联软删 links+batches+label
  出参 `ApiResponse<{deleted_count}>`("可用链接"二次确认由前端按 link_count 弹)

### B. 群链接
- **B1** `POST /api/group-links/import` — 导入(multipart)
  入参:`file`(可选 TXT/CSV/Excel)+ 表单 `label_id`/`batch_name`/`text`(可选);file/text 至少一个
  行为(同步):解析→行→逐行 upsert-by-url→写 batch+detail
  出参 `ApiResponse<GroupLinkImportResultVO>`:`{batch_id, total, inserted, adopted, duplicated, failed, errors:[]}`
- **B2** `GET /api/group-links` — 分组下链接(编辑/分组信息视图)
  入参 Query:`label_id`(必)/`keyword`/`page`/`page_size`
  出参 `PageResult<GroupLinkVO>`:`{id, url, group_name, source_file_name(JOIN batch), created_at}`
- **B3** `POST /api/group-links/migrate` — 迁移分组 `{link_ids:[], target_label_id}` → `ApiResponse<{migrated_count}>`
- **B4** `POST /api/group-links/batch-delete` — 删链接(链接级)`{ids:[]}` → `ApiResponse<Void>`(软删)

### C. 导入明细 / 导出
- **C1** `GET /api/group-link-imports/details` — 明细
  入参 Query:`label_id` 或 `batch_id` / `result`(可空=全部)/ `page`/`page_size`
  出参 `PageResult<GroupLinkImportDetailVO>`:`{line_no, group_name, raw_url, source_file_name, result, fail_reason, created_at}`
- **C2** `GET /api/group-link-imports/failed/export` — 导出失败(CSV)
  入参 Query:`label_id` 或 `batch_id`;出参 `text/csv`(行号,群名,链接,失败原因,导入时间);读 detail where result≥3

## 7. 导入流程(同步)

```
multipart ─→ FileLinesExtractor(TXT纯文本 / CSV+Excel走FastExcel) ─→ List<String> ∪ 手填text
          ─→ 建 batch ─→ LineImporter(结构化逐行产出) ─→ 逐行 upsert-by-url:
                          · 新 url            → insert group_link + detail(SUCCESS)
                          · 已存在/软删(异分组)→ revive+改label_id + detail(ADOPTED)
                          · 批内重复           → detail(DUPLICATE)
                          · 格式错             → detail(FORMAT_ERROR)
          ─→ 回写 batch 计数 ─→ 返回 GroupLinkImportResultVO
```

- **url 归一化**:去 scheme / 末尾斜杠 / host 小写,邀请码保留大小写;归一化串入库 + 去重。
- **`LineImporter` 演进(反屎山 #2:改调用方不加并行重载)**:现版 `run` 只返回聚合 Result + 错误字符串,喂不了逐行 detail(行号/原文/结果)。重构为**返回结构化逐行结果**为核心(`List<LineOutcome>{lineNo, raw, status, reason, record}`),聚合统计由其汇总;**同步更新 IP 块调用方**(有 `LineImporterTest`/`IpProxyServiceImplTest` 兜底)。理由=本块当下需逐行 detail,非为 account 提前抽象。

## 8. 传输对象 + 依赖

- **对象**(`group/model/`):entity 普通类(`GroupLinkLabel`/`GroupLink`/`GroupLinkImportBatch`/`GroupLinkImportDetail`)、`*Query` 可变 class extends PageQuery、`*DTO`/`*VO` record;转换用 `GroupConverter`(MapStruct);无 Repository,Service 直调 Mapper。
- **时间口径**:库 `DATETIME`(UTC),entity `LocalDateTime`,VO 出参 `Long` epoch 毫秒(converter `toInstant(ZoneOffset.UTC).toEpochMilli()`,对齐 `MarketingTemplateConverter`)。
